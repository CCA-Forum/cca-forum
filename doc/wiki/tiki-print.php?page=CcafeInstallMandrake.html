Mandrake ccaffeine installation issues
Aug 24, 2005 (baa)
0.10.9 is not a supported babel release.
Fixing dccafe/cxx/Makefile.Rules.in to support 0.10.9 by
adding the appropriate defines needed may improve your mileage.
Search for 10.6 to find the relevant block in the makefile.
After that, configure dccafe with
--with-csb-config=/home/norris/cca/dev/bin/cca-spec-babel-config
and verify that home/norris/cca/dev/bin/cca-spec-babel-config
is a symlink to the right version.

Aug. 12, 2005 (bn)

I should point out that this is a description of an unsuccessful installation attempt on a couple of Mandrake 10.1 machines. The result of 'uname -a' on one of them is

Linux norris-r40-10.mcs.anl.gov 2.6.11-acpi-perfctr #2 Wed Jun 15 18:38:27 CDT 2005 i686 Mobile Intel(R) Pentium(R) 4 - M CPU 2.20GHz unknown GNU/Linux

The prerequisites were CVS or SVN development versions, cleanly checked out on Aug 12, 2005:

  • babel (0.10.9), configured with:
./configure --enable-shared --disable-static --enable-fortran90=/opt/intel_fc_81/bin/ifort --with-chasm=/home/norris/cca --prefix=/home/norris/cca/dev

  • cca-spec-babel (0.7.8), configured with:
./configure --prefix=/home/norris/cca/dev --with-babel-config=/home/norris/cca/dev/bin/babel-config --enable-shared --with-boost=/home/norris/software/boost_cvs

  • cca-spec-classic (0.5.7), configured with:
./configure --prefix=/home/norris/cca/dev --with-babel-libtool=/home/norris/cca/dev/bin/babel-libtool

  • ccaffeine (from cvs), configured with:
./configure --with-prefix=/home/norris/cca/dev --enable-shared --with-mpi=no --with-classic-config=/home/norris/cca/dev/bin/cca-spec-classic-config --with-csb-config=/home/norris/cca/dev/bin/cca-spec-babel-config --without-lapack --with-boost=/home/norris/software/boost_cvs


The configuration logs, as well as the environment settings are attached to this page. The results of the Ccaffeine tests:
